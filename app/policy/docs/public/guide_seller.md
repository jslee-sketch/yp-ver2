# 판매자 가이드

## 역핑이란?
역핑은 소비자 주도형 역경매 플랫폼입니다.
구매자가 원하는 제품/가격을 제시하면 판매자가 오퍼를 제출합니다.
경쟁력 있는 오퍼를 제출하여 거래를 성사시키세요.

---

## 회원가입 / 로그인
이메일 또는 소셜(카카오/네이버/구글) 로그인으로 가입합니다.
회원가입 시 "판매자" 역할을 선택합니다.
소셜 로그인 시에도 역할 선택 → 닉네임 설정 → 완료 후 자동 로그인됩니다.

### 판매자 등록
1. 회원가입 → 판매자 역할 선택
2. 사업자 정보 입력: 상호명, 사업자번호, 대표자명
3. 서류 업로드: 사업자등록증, 통신판매업신고증(선택), 통장사본
4. 관리자 승인 대기 (보통 1-2일)
5. 승인 완료 → 오퍼 제출 가능
> 자주 묻는 표현: 판매자 승인, 판매자 가입, 사업자등록, 판매자 등록, 승인 대기

## 판매자 대시보드
경로: /seller
총 오퍼 수, 낙찰률, 매출, 정산 현황을 한 눈에 확인합니다.
**판매자 레벨(Lv.1~Lv.6)**과 **현재 수수료율**이 대시보드 헤더에 표시됩니다.
기간별 매출 추이, 오퍼 채택률, 평균 배송 소요일, 리뷰 평점 추이, 환불률도 확인 가능합니다.
> 자주 묻는 표현: 내 레벨, 판매자 레벨, 수수료율 확인, 대시보드 레벨

## 딜 탐색
경로: /deals
현재 열린 딜 목록을 카테고리/키워드로 검색합니다.
제품명, 목표가, 수량, 요청사항을 확인합니다.

---

## 오퍼 제출
딜 상세에서 가격, 배송비, 배송기간, 조건을 입력하여 오퍼를 제출합니다.
오퍼 마감 시간은 48시간입니다. (defaults.yaml 기준)

### 오퍼 작성 항목
| 항목 | 설명 |
|------|------|
| 단가 | 개당 판매 가격 |
| 수량 | 공급 가능 수량 |
| 배송비 | 배송비 (무료배송 시 0) |
| 배송 방식 | 택배/직거래/무료배송 |
| 환불 조건 | 환불 가능 여부 및 상세 조건 |
| 유효기간 | 오퍼 만료 기한 (기본 48시간) |

### 오퍼 전략 팁
- 공동구매 참여자가 많을수록 자동 가격 인하 적용
- 배송비 무료 오퍼가 선택률이 높음
- 환불 조건이 유연할수록 구매자 신뢰 상승

## 오퍼 관리
경로: /seller/offers
제출한 오퍼 목록에서 **오퍼 번호(O-{id})**, **딜 번호(D-{deal_id}, 클릭 이동)**, **상품명**, 오퍼 가격(콤마+원), 배송비, 상태(색상 배지), 제출일을 확인합니다.
구매자 결제 전까지 수정/철회 가능합니다.
> 자주 묻는 표현: 오퍼 수정, 오퍼 철회, 가격 변경, 오퍼 가격 수정, 오퍼 마감 시간, 오퍼 마감

---

## 배송 관리
경로: /seller/delivery
택배사 선택 + 운송장 번호 입력으로 발송 처리합니다.
각 배송 건에 **예약 #{id}**, **딜 #{deal_id}(클릭하면 딜 상세로 이동)**, 구매자명, 상품명, 배송 상태(색상 배지), 택배사/운송장, 발송일이 표시됩니다.
스마트택배(SweetTracker) API로 실시간 배송 상태가 자동 추적됩니다.
배달 완료 후 3일 내 구매자가 수취 확인하지 않으면 자동 구매확정됩니다.

### 배송 처리 주의사항
- 결제 확인 후 **48시간 이내** 배송 처리 권장
- 3일 초과 시 관리자 모니터링 대상
- 운송장번호 오입력 시 수정 가능
- 배송 상태: 배송준비 → 집하 → 이동중 → 배달중 → 배달완료

> 지원 택배사: CJ대한통운, 한진택배, 로젠택배, 우체국택배, 롯데택배 등
> 자주 묻는 표현: 운송장 입력, 배송 처리 방법, 택배사 선택, 운송장 번호 여러 개, 배송 처리, 발송 처리

---

## 배송 정책
경로: /seller/shipping-policy
배송비 유형 설정:
- **무료배송 (FREE)**: 배송비 0원
- **건당 배송비 (PER_RESERVATION)**: 예약 건당 고정 배송비
- **수량당 배송비 (PER_ITEM)**: 기본배송비 + (개당배송비 × 수량)
제주/도서산간 추가 배송비, 교환/반품 배송비도 설정 가능합니다.
> 자주 묻는 표현: 배송비 설정, 수량당 배송비, 건당 배송비, 무료배송, 배송비 유형

## 정산 관리
경로: /seller/settlements
정산 내역에서 **정산 번호(S-{id})**, **예약 #{reservation_id}(클릭 이동)**, **딜 #{deal_id}(클릭 이동)**, 총액/수수료/정산액, 상태(색상 배지), 생성일/지급일을 확인합니다.
총 정산액 합계와 건수 요약이 상단에 표시됩니다.
**PDF 다운로드** 버튼으로 정산 내역서를 다운로드할 수 있습니다.
구매확정 후 정산 파이프라인을 거칩니다.

### 정산 흐름
```
HOLD (쿨링 7일) → READY → APPROVED (관리자 승인) → PAID (입금)
```

### 정산 금액 계산
```
정산액 = 결제금액 - PG수수료(3.3%) - 플랫폼수수료(레벨별)
```

### 플랫폼 수수료 (레벨별)
| 레벨 | 거래 건수 | 최소 평점 | 수수료율 |
|------|----------|----------|---------|
| Lv.6 (신규) | 0-20 | - | 3.5% |
| Lv.5 | 21-40 | 4.0 | 3.0% |
| Lv.4 | 41-60 | 4.0 | 2.8% |
| Lv.3 | 61-100 | 4.0 | 2.7% |
| Lv.2 | 101+ | 4.0 | 2.5% |
| Lv.1 | 101+ | 4.5 | 2.0% |

> 분쟁 중인 거래는 정산이 자동 보류됩니다.
> 정산 지급일: 약 30일 (쿨링 7일 + 관리자 승인)
> 자주 묻는 표현: 정산 언제, 쿨링 기간, 수수료율, 수수료 레벨, 정산 금액, 정산 쿨링

## 반품/교환 관리
경로: /seller/returns
반품 요청 확인, 승인/거부 처리.
- **refund**: 단순 환불 (배송 전)
- **return**: 반품 (배송 후, 상품 회수 필요)
- **exchange**: 교환 (새 상품 재배송)

---

## 환불 관리
경로: /seller/refunds
환불 요청 목록에서 예약 번호(클릭 이동), 구매자명, **환불 사유**, 환불 금액, 상태(색상 배지)를 확인합니다.
동의/미동의 처리가 가능합니다.
환불 시뮬레이터로 환불 금액을 미리 계산할 수 있습니다.
> 자주 묻는 표현: 환불 요청 대응, 환불 동의, 환불 거부, 구매자 환불 요청, 환불 시뮬레이터

### 환불 유형별 차이
- **구매자 사유**: 배송 시작 후 배송비 환불 불가. PG수수료(3.3%) 구매자 부담.
- **판매자 사유**: 배송비/수수료 판매자 부담.
- **시스템 오류**: 배송비/수수료 시스템 부담.
- **분쟁 결과**: 배송비 항상 환불. 수수료 시스템 부담.

## 분쟁 대응
구매자가 분쟁을 접수하면:
1. 알림 수신 → 정산 관리에서 확인
2. 관리자가 양측 의견 확인
3. 증빙 자료 제출 (배송 증빙, 상품 사진 등)
4. 관리자 판정: 환불/부분환불/기각
5. 결과에 따라 정산 조정
6. 분쟁 종료 후 3일 뒤 정산 재개

> 분쟁이 자주 발생하면 레벨 하락 및 계정 제한 가능

## 세금계산서

각 세금계산서에 송장번호, **정산 번호 링크(S-{id}, 클릭하면 정산 상세로 이동)**, 공급가액/세액/합계(콤마+원), 상태(색상 배지), 발행일이 표시됩니다.
공급자: (주)텔러스테크 (플랫폼 운영사).
PENDING(확인 대기) 건의 [확인] 버튼을 눌러 내용을 확인하세요.
확인 후 관리자가 일괄 발행합니다.
> 자주 묻는 표현: 세금계산서, 세금계산서 확인, 세금계산서 발행, 세금계산서 어디서

## 자주 묻는 질문
- **Q: 오퍼 수정 가능?** → 구매자 결제 전까지 수정/철회 가능
- **Q: 정산 언제?** → 쿨링(7일) + 관리자 승인 후 지급 (약 30일)
- **Q: 분쟁 보류 해제?** → 분쟁 종료 후 3일 후 정산 재개
- **Q: 수수료 인하?** → 거래 실적+평점 쌓아서 레벨업하면 자동 인하
- **Q: 액추에이터?** → 판매자 모집/관리하는 중간자, 커미션 별도 정산
- **Q: 배송 추적?** → 운송장 입력 후 스마트택배 API로 자동 추적
- **Q: 배송비 설정?** → /seller/shipping-policy에서 무료/건당/수량당 설정
- **Q: 세금계산서 어디서 봐?** → /seller/tax-invoices에서 확인
- **Q: 세금계산서 확인 방법?** → PENDING 건의 [확인] 버튼 → 관리자 일괄 발행
- **Q: 사업자 정보 수정?** → /seller/business-info에서 수정 (OCR 자동 입력 지원)
