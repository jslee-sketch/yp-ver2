# 구매자 가이드 (Public)

## 구매자 이익(핵심)
- 원하는 가격/조건을 먼저 제시 → 판매자 경쟁 유도
- 오퍼 비교로 가격+배송+환불+조건 동시 판단 → 선택 품질 향상
- 신뢰/분쟁/환불/정산 체계로 거래 리스크 감소

## 핑퐁이에게 물어보는 팁
- 확정 답: “예약#403 환불 얼마야?”, “오퍼#4 배송비?”
- 원칙 질문: “환불은 어떤 경우 가능해?”, “Dead Time이 뭐야?”

## 딜 검색/필터 기능

원하는 딜을 빠르게 찾을 수 있습니다.

| 필터 | 설명 |
|------|------|
| **상태** | 모집 중(open) / 마감(closed) / 완료(completed) |
| **키워드** | 제품명 검색 (예: “에어팟”, “갤럭시”) |
| **희망가 범위** | 최소~최대 가격 필터 |
| **정렬** | 최신순 / 희망가순 / 마감 임박순 |
| **내 딜만** | 내가 만든 딜만 보기 |

```
GET /deals/?status=open&keyword=에어팟&sort=deadline_at:asc&page=1&size=20
```

## 신고 접수

허위 오퍼, 사기, 부적절한 게시글 등은 **신고**를 통해 접수할 수 있습니다.

```
POST /reports
Body: {
  “target_type”: “reservation”,   // deal / offer / reservation / user
  “target_id”: 403,
  “reason_code”: “fraud”,         // fraud / fake_offer / abuse / spam / other
  “description”: “배송 안 됐는데 완료로 처리됨”
}
```

**신고 처리 흐름**:
1. 신고 접수 → `PENDING`
2. 관리자 검토 → `REVIEWED`
3. 처리 완료 → `RESOLVED` (결과 알림) 또는 `DISMISSED` (무혐의)

> 허위 신고 반복 시 계정 정지 대상이 될 수 있습니다.

## 회원 탈퇴 절차

1. **진행 중인 예약 정리**: 결제/배송 중인 예약이 없는지 확인
2. **탈퇴 신청**: `DELETE /account/withdraw`
3. **탈퇴 완료**: `is_active = False` 처리 (즉시 로그인 불가)
4. **개인정보 처리**: 30일 후 자동 익명화

**탈퇴 시 주의사항**:
- 보유 포인트는 탈퇴 즉시 소멸됩니다
- 작성한 코멘트/예측은 “탈퇴한 사용자”로 익명 처리됩니다
- 진행 중인 예약이 있으면 탈퇴가 거부됩니다 (예약 정리 후 재신청)
