# 2. 가격 축의 정의

역핑에는 4개의 가격 축이 존재한다.

## 2.1 Target Price (목표가)

- 딜방 참여자들이 협의하여 최종 입력한 가격
- 집단지성의 결과
- 외부 참고자료(URL/사진/시장가격 등)를 기반으로 합의된 값
- 시스템이 자동 생성하는 값이 아님
- 최종 입력값이며 딜 생성의 기준점

Target은 "할인된 기대치"가 반영될 수 있음.

---

## 2.2 Anchor Price (역핑 기준가)

- 역핑이 외부 시장 데이터, 내부 데이터, 조건 등을 기반으로 산정한 가격
- 비동기 계산 가능
- Target과 별개로 존재
- Guardrail 판단에 사용됨

Anchor는 시장 현실 축이다.

---

## 2.3 Group Price (공동구매가)

- 계산식: G(Q_offer) 기반
- Q_offer = min(Q_room, offer_cap)

여기서:
- Q_room = 딜방 목표 수량(qty_target)
- offer_cap = 해당 판매자의 판매 가능 수량
- Q_offer = 해당 오퍼가 실제로 소화 가능한 수량

공동구매가는 "딜방 전체 수량"이 아니라
"해당 오퍼가 감당 가능한 수량" 기준으로 계산한다.

이것은 현실 왜곡 방지를 위한 원칙이다.

---

## 2.4 Seller Offer Price (판매자 제시가)

- 판매자가 입력한 고정값
- 역핑은 이 값을 변경하지 않는다
- 모든 비교는 이 값을 기준으로 수행된다

---

# 3. 비교 규칙

## 3.1 기본 비교 축

랭킹 화면에서는 다음 4개만 사용한다:

1. 판매가 (offer_price)
2. 목표가 대비 % (offer_price / target_price * 100)
3. 남은수량 (remaining_qty)
4. 그룹 (PREMIUM / MATCHING / BELOW)

목표가 대비 %는 구매자 관점 직관 축이다.

---

## 3.2 상세 비교 (Preview Pack)

Preview에서는 다음 비교를 제공한다:

- 기준조건 대비
- 공동구매 기준 대비

단,
판매자 가격은 절대 수정하지 않는다.

---

# 4. 문장 표현 규칙

## 4.1 금지사항

- 음수 표기 금지
- "-6500원" 같은 표현 금지

## 4.2 표현 규칙

- "기준조건 대비 3,200원 저렴"
- "공동구매 기준 대비 유사"

유사 판정 기준:
- 차이가 매우 작은 경우 "유사"

---

# 5. 더블 할인 방지 원칙

Target은 이미 할인 기대가 반영된 값일 수 있다.

따라서:
- 공동구매 계산 시 Target을 직접 할인 기준으로 사용하지 않는다.
- Group Price는 Anchor/기준조건 축을 기반으로 계산한다.

---

# 6. Guardrail 원칙

딜 생성 시:

만약

Target이 Anchor 대비 일정 범위를 크게 벗어날 경우:

- 경고 표시
- 재협의 요청
- 또는 딜 생성 제한 가능

Guardrail은 시장 왜곡 방지 장치이다.

---

# 7. UX 계층 분리

## 7.1 1단계 화면 (랭킹)

단순 비교:
- 판매가
- 목표가 대비 %
- 남은수량
- 그룹

## 7.2 2단계 화면 (상세)

- 기준조건 대비
- 공동구매 기준 대비
- 환산 설명
- 조건 영향 설명

정보는 단계적으로 제공한다.

---

# 8. 철학 요약

역핑은:

- 판매자 가격을 조작하지 않는다
- 집단지성을 존중한다
- 시장 현실을 무시하지 않는다
- 비교 기준을 제공할 뿐이다
