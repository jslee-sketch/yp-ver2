## 예약/주문 관리
경로: /admin/reservations
상태 드롭다운 (다크모드), 상태 설명:
- PENDING: 결제 대기 (5분 제한)
- PAID: 구매자 결제 완료
- SHIPPED: 판매자 발송 완료
- CONFIRMED: 구매자 수취 확인 또는 자동 구매확정
- CANCELLED: 취소됨
- REFUNDED: 환불됨
- DISPUTED: 분쟁 중

## 배송 관리
경로: /admin/delivery
배송 상태 요약 카드 (전체발송/집하/이동중/배달중/완료/수취대기/자동확정대기).
각 배송 건에 **예약 번호(R-{id}, 클릭하면 예약 상세로 이동)**, 판매자/구매자명, 배송 상태(색상 배지), 택배사/운송장이 표시됩니다.
[🔍 일괄 배송 조회]: 배송 중인 모든 주문 상태 업데이트 (스마트택배 API, 일 100건 무료).
[✅ 자동 구매확정 실행]: 배달완료 후 3일 경과 주문 자동 확정.
배송 추적 간격: 30분 (defaults.yaml 기준).
> 자주 묻는 표현: 일괄 배송 조회, 자동 구매확정, 배달 완료 후, 스마트택배

## 정산 관리
경로: /admin/settlements
HOLD → READY → APPROVED → PAID 파이프라인.
**정산 번호(S-{id})**, **예약 번호(R-{id}, 클릭 이동)**, 판매자명, 결제액/수수료/정산액, **생성일** 컬럼이 표시됩니다.
상태 드롭다운 필터로 상태별 조회 가능.
**단건 승인** 버튼과 **일괄 승인** 버튼으로 READY → APPROVED 처리.
**분쟁 플래그(⚠️)**: 분쟁 중인 거래에 경고 배지 표시.
쿨링 기간(7일) 후 자동으로 READY 전환.
관리자가 APPROVED 처리 → 입금 확인 후 PAID.
분쟁 중인 거래는 자동 보류.
정산 지급 지연: 30일 (defaults.yaml 기준).
> 자주 묻는 표현: 일괄 승인, 정산 승인, 분쟁 플래그, 정산 생성일

## 세금계산서 관리
경로: /admin/tax-invoices
정산 APPROVED 시 세금계산서 자동 생성 (PENDING).
각 세금계산서에 송장번호, **정산 번호(S-{id})** 컬럼, 판매자명, 공급가액/세액/합계가 표시됩니다.
상태 파이프라인: PENDING → CONFIRMED(판매자 확인) → ISSUED(발행) / CANCELLED(취소).
공급자: (주)텔러스테크 (고정, defaults.yaml tax_invoice.supplier).
탭별 필터: 전체/PENDING/CONFIRMED/ISSUED/CANCELLED.
체크박스 선택 → [일괄 발행] / [ECOUNT 내보내기].
행 클릭 → 상세 모달 (공급자/공급받는자 정보, 금액 상세).
ECOUNT 엑셀 내보내기: 선택한 건 또는 전체를 XLSX로 다운로드.

## 환불 시뮬레이터
경로: /admin/refund-simulator
**수동 모드**: 상품단가/수량/배송비/배송상태/환불사유/정산상태를 입력 → 환불 결과 미리 계산.
**예약 조회 모드**: 예약 번호 입력 시 해당 예약의 환불 프리뷰.

### 배송비 3가지 모드
- **무료배송**: 배송비 0원
- **건당 배송비**: 예약 건당 고정 배송비
- **수량당 배송비**: 기본배송비 + (개당배송비 × 수량)

### 환불 사유 통합
- **구매자 사유**: 배송 시작 후 배송비 환불 불가, PG수수료(3.3%) 구매자 부담
- **판매자 사유**: 배송비/수수료 판매자 부담
- **시스템 오류**: 배송비/수수료 시스템 부담
- **분쟁 결과**: 배송비 항상 환불, 수수료 시스템 부담
> 자주 묻는 표현: 환불 시뮬레이터, 환불 계산, 배송비 모드, 환불 미리보기, 수동 모드, 예약 조회 모드

## 마이너리티 리포트
경로: /admin/minority-report
사용자 행동 분석 대시보드.
구매자 8개 + 판매자 14개 = 22개 행동 수집 포인트.

### 구매자 행동 수집 (8개)
검색, 딜 조회, 가격여정 조회, 딜 참여, 핑퐁이 대화, 카테고리 탐색, 브랜드 관심, 관전 모드

### 판매자 행동 수집 (14개)
검색, 딜 조회, 오퍼 제출/수정, 배송 처리, 정산 확인, 환불 처리, 리뷰 답글, 문의 답변, 정책 변경, 배송 속도, 대시보드 조회, 통계 조회, 수수료 확인, 공지 확인

### AI 프로파일링
- **구매자**: 관심 카테고리/브랜드/가격대/구매의지/행동패턴 분석
- **판매자**: 강점카테고리/가격패턴/낙찰률/배송신뢰도/리뷰응대/위험도 분석
- **망설이는 구매자 감지** → [넛지] 알림 발송
- **판매자 오퍼 기회 매칭** → 자동 추천
- 인기 검색 키워드, 판매자 스킵 패턴 분석
> 자주 묻는 표현: 마이너리티 리포트, 행동 분석, 행동 수집 포인트, 넛지 알림, 망설이는 구매자, 스킵 패턴, 프로파일링, 오퍼 기회 매칭

## 가격 합의 엔진
딜 생성 시 3중 소스(네이버쇼핑 + 쿠팡 + GPT-4o) 교차 검증으로 시장가 산출.
신뢰도 등급: 🟢높음(15% 이내 일치) / 🟡보통(30% 이내) / 🔴낮음 / ⚫온라인 미판매.

### 자동 필터링
- 모델 정확 매칭: "S23" 검색 시 S24/S25/S26 자동 제외 (그램16/스위치2/워치7/OLED 등 21개 패턴)
- 브랜드 필터링: 13개 브랜드 한/영 별칭 자동 매핑
- 액세서리/묶음/중고 자동 제외 (검색어 자체에 "세트" 포함 시 제외하지 않음)
- 고가 제품(자동차/부동산) 감지 → 온라인 미판매 안내
- 명품(롤렉스/샤넬 등 14개 브랜드) → 가품 주의 경고

## 알림 관리
경로: /admin/notifications
전체 알림 목록, 브로드캐스트 알림 발송.
FCM 푸시 알림 토큰 등록 현황 확인.

### 알림 시스템 개요
- 총 64개 이벤트 (구매자 22 + 판매자 22 + 액추에이터 13 + 관리자 7)
- 채널: 앱 내 알림 + FCM 푸시 + 이메일 (이메일은 placeholder)
- 사용자별 이벤트 채널 ON/OFF 설정 가능
- 관심 상품 매칭: 딜 생성 시 관심 등록 사용자에게 자동 알림

### 관리자 전용 알림 이벤트
- NEW_SELLER_PENDING: 판매자 승인 대기
- NEW_DISPUTE: 새 분쟁 접수
- NEW_REPORT: 새 신고 접수
- SETTLEMENT_BATCH_READY: 정산 일괄 승인 대기
- TAX_INVOICE_BATCH_READY: 세금계산서 일괄 발행 대기
- ANOMALY_DETECTED: 이상 감지
- SYSTEM_ERROR: 시스템 오류
> 자주 묻는 표현: 관리자 알림, 알림 설정, 이상감지 알림, 분쟁 알림
