# 딜 상세 화면 (Deal Detail)

> 최종 수정: 2026-02-28  
> 우선순위: ★★★★★ (역핑의 얼굴)  
> 역할: 가격 투명성 + 오퍼 경쟁 + 관전자 게임화가 한 화면에 집약

---

## 1. 진입 경로

| 출발지 | 방법 |
|--------|------|
| 홈 피드 | 딜 카드 탭 |
| 딜 검색 | 검색 결과 카드 탭 |
| 알림 | "새 오퍼 도착" / "관전 결과" 알림 탭 |
| 핑퐁이 | 딜 관련 대화 → 딥링크 |
| 마이 딜 | 히스토리에서 탭 |
| 외부 공유 | URL 딥링크 (yeokping://deal/{id}) |

---

## 2. 사용하는 API

| API | 용도 | 호출 시점 |
|-----|------|----------|
| GET /deals/{id} | 딜 기본 정보 (상품명, 브랜드, 목표가, anchor, 상태, 마감시간) | 화면 진입 |
| GET /offers/deal/{id}/ranked | 오퍼 목록 (가격순 + 티어 그룹핑) | 화면 진입 |
| GET /spectator/predictions/{deal_id}/count | 관전자 수 + 예측 분포 | 화면 진입 |
| GET /spectator/predictions/{deal_id}/mine | 내 예측 여부/결과 | 화면 진입 (로그인 시) |
| GET /deals/{id}/participants | 참여자 목록/수 | 화면 진입 |
| GET /deal-chat/{deal_id} | 딜방 채팅 메시지 | 채팅 섹션 스크롤 시 |
| POST /spectator/predict | 관전자 예측 제출 | 예측 버튼 탭 |
| POST /v3_6/reservations | 예약 생성 | 오퍼 선택 → 예약 |
| GET /delivery/track/{reservation_id} | 배송 추적 | 배송 중 상태일 때 |

## 3. 화면 구성 (위→아래 순서)

### 3-1. 상단 바

```
[← 뒤로]              [🔔 알림]  [⋮ 더보기]
```

- 뒤로: 이전 화면
- 알림: 읽지 않은 수 배지
- 더보기: 공유/신고/관심 딜 등록

### 3-2. 딜 헤더

```
● LIVE                           23시간 남음
에어팟 프로 2세대 (USB-C)
Apple · 전자기기 · 참여자 12명      👀 87명 관전 중
```

데이터 매핑:
- 상태 배지: deal.status → OPEN이면 "LIVE" (녹색), CLOSED면 "마감" (회색)
- 마감 시간: deal.deadline_at → 현재 시간과 차이 계산 ("23시간 남음")
- 상품명: deal.product_name
- 브랜드: deal.brand (AI 자동 추출)
- 참여자 수: participants.length
- 관전자 수: spectator_count

### 3-3. 가격 대시보드 (핵심 영역)

```
┌─────────────────────────────────────────────┐
│  📊 시장가 (네이버)    🎯 목표가    ⚡ 최저 오퍼  │
│     103,900원          95,000원      89,000원  │
│                                               │
│  ┌─────────────────────────────────────────┐  │
│  │  🔥 시장가 대비 14,900원 절약 (14%↓)    │  │
│  └─────────────────────────────────────────┘  │
│                                               │
│  목표 달성률              93%                  │
│  ████████████████████░░░                      │
└─────────────────────────────────────────────┘
```

데이터 매핑:
- 시장가: deal.anchor_price (네이버 최저가 기반)
- 목표가: deal.desired_price (구매자 희망가)
- 최저 오퍼: offers[0].price (ranked 결과의 1위)
- 절약 금액: anchor_price - lowest_offer_price
- 절약 퍼센트: (절약 금액 / anchor_price) × 100
- 달성률: (lowest_offer / desired_price) × 100 (100% 이하면 목표 초과 달성)

가격 표시 규칙:
- anchor_price가 null이면 → "시장가 분석 중..." 표시 (AI 비동기 계산 대기)
- anchor_price 도착 시 → 애니메이션과 함께 가격 표시
- 최저 오퍼가 목표가보다 낮으면 → "목표 초과 달성!" 녹색 강조
- 최저 오퍼가 목표가보다 높으면 → 달성률 표시 + "아직 경쟁 중"

### 3-4. 관전자 예측 분포

```
┌─────────────────────────────────────────────┐
│  🎯 관전자 예측 분포 (87명)                   │
│                                               │
│  85-88K  ██░░░░░░░░░░░░░  12%               │
│  88-91K  █████████░░░░░░  35%  ← 최다       │
│  91-94K  ███████░░░░░░░░  28%               │
│  94-97K  █████░░░░░░░░░░  18%               │
│  97-100K ██░░░░░░░░░░░░░   7%               │
│                                               │
│  평균 예측가: 92,000원                        │
│                                               │
│  [ 🎯 나도 예측하기 ]    ← 관전자가 아직 미예측 시 │
│  ✅ 내 예측: 91,000원    ← 이미 예측한 경우     │
└─────────────────────────────────────────────┘
```

데이터 매핑:
- 예측 분포: spectator/predictions/{deal_id}/count → distribution 배열
- 평균 예측가: spectator/predictions/{deal_id}/count → avg_predicted_price
- 내 예측: spectator/predictions/{deal_id}/mine → predicted_price (null이면 미예측)
- 예측 버튼: POST /spectator/predict (deal_id, predicted_price, buyer_id)

예측 입력 UX:
- 버튼 탭 → 바텀 시트 오픈
- 가격 슬라이더 (anchor_price ±30% 범위)
- 직접 입력도 가능
- 코멘트 (선택, 최대 200자)
- 제출 후 → 분포 차트에 내 위치 하이라이트

관전 결과 (딜 종료 후):
- 성사가 vs 내 예측 비교
- 오차율 + 판정 티어 (PERFECT/EXCELLENT/GOOD/FAIR/MISS)
- 획득 포인트
- "다음 딜도 관전하기" CTA

### 3-5. 오퍼 목록

```
오퍼 4개 경쟁 중

┌─ PREMIUM ────────────────────────────────┐
│ [PREMIUM] 디지털플러스        89,000원    │
│ ⭐ 4.8 (342)    무료배송 · 2일 · 1년보증  │
│                          [ 선택하기 ]     │
├──────────────────────────────────────────┤
│ [PREMIUM] 테크마켓            91,000원    │
│ ⭐ 4.6 (215)    무료배송 · 3일 · 6개월    │
│                          [ 선택하기 ]     │
├─ MATCHING ───────────────────────────────┤
│ 애플매니아                    93,500원    │
│ ⭐ 4.9 (891)    무료배송 · 1일 · 1년보증  │
│                          [ 선택하기 ]     │
├─ BELOW ──────────────────────────────────┤
│ 전자왕                        96,000원    │
│ ⭐ 4.2 (67)     3,000원 · 4일 · 보증없음  │
│                          [ 선택하기 ]     │
└──────────────────────────────────────────┘
```

데이터 매핑:
- 오퍼 목록: GET /offers/deal/{id}/ranked → 티어별 그룹핑
- 각 오퍼: id, seller_name, price, tier, rating, review_count, shipping_fee, delivery_days, warranty_months
- PREMIUM: accent border + gradient bg, 배지 표시
- MATCHING: 기본 카드
- BELOW: muted 스타일

오퍼 카드 탭 → 오퍼 상세 바텀 시트:
- 셀러 상세 (레벨, 클레임율, 평균 배송일)
- 조건 비교 표 (배송비, 배송일, 보증, 반품조건)
- 핑퐁이 코멘트 ("이 오퍼는 배송이 빠르지만 보증기간이 짧아요")
- [예약하기] 버튼

### 3-6. 핑퐁이 인사이트 카드

```
┌─────────────────────────────────────────────┐
│  🤖 핑퐁이                                   │
│                                               │
│  현재 최저 오퍼가 시장가보다 14% 저렴해요.      │
│  관전자 87명의 평균 예측가는 92,000원이고,      │
│  최저 오퍼(89,000원)는 예측 평균보다도 낮아요.  │
│  충분히 좋은 딜이에요! 👍                      │
│                                               │
│           [ 핑퐁이에게 질문하기 ]              │
└─────────────────────────────────────────────┘
```

핑퐁이 자동 생성 문구 (상황별):

| 상황 | 문구 템플릿 |
|------|-----------|
| 최저 오퍼 < 목표가 | "목표가보다 {차이}원 더 저렴한 오퍼가 있어요! 🎉" |
| 최저 오퍼 ≈ 목표가 (±5%) | "목표가에 거의 근접한 오퍼가 있어요. 지금 선택하면 좋을 것 같아요." |
| 최저 오퍼 > 목표가 | "아직 목표가에 도달한 오퍼가 없어요. 시간이 {남은시간} 남았으니 조금 더 기다려보세요." |
| PREMIUM 오퍼 2개 이상 | "PREMIUM 오퍼가 {N}개나 경쟁 중이에요! 조건을 비교해보세요." |
| 관전자 예측 평균 < 최저 오퍼 | "관전자들은 더 낮은 가격을 예상하고 있어요. 오퍼가 더 들어올 수 있어요." |
| 관전자 예측 평균 > 최저 오퍼 | "최저 오퍼가 관전자 예측 평균보다 낮아요. 좋은 기회예요!" |
| 오퍼 0개 | "아직 오퍼가 없어요. 판매자들이 준비 중이에요 — 조금만 기다려주세요! ⏳" |
| 마감 임박 (3시간 이내) | "⚡ 마감까지 {남은시간}! 지금 결정하지 않으면 놓칠 수 있어요." |

"핑퐁이에게 질문하기" 탭 → 핑퐁이 채팅 바텀 시트 오픈

### 3-7. 딜방 채팅 (접힘 가능)

```
💬 딜방 채팅 (3)                    [펼치기 ▼]
──────────────────────────────────────────
 buyer_01: 무료배송 아닌 오퍼도 있나요?
 buyer_03: 디지털플러스 1년보증 괜찮은듯
 buyer_01: ㅇㅇ 그거 선택하려구
```

데이터: GET /deal-chat/{deal_id}
- 기본 접힘 상태, 탭하면 펼침
- 최신 3개 미리보기
- 펼치면 전체 채팅 + 입력 필드

## 4. 상태별 화면 변화

### 4-1. OPEN (진행 중)

- 가격 대시보드: 전체 표시
- 오퍼 목록: 활성, [선택하기] 버튼 활성
- 관전자: 예측 참여 가능
- 핑퐁이: 실시간 인사이트

### 4-2. CLOSED (마감 — 오퍼 선택 완료)

- 상태 배지: "마감" (회색)
- 성사가 표시: "성사가: 89,000원"
- 오퍼 목록: 선택된 오퍼 하이라이트, 나머지 dimmed
- 관전자: 예측 결과 공개 (내 예측 vs 성사가)
- 핑퐁이: "이 딜은 시장가 대비 14% 저렴하게 성사되었어요!"

### 4-3. 배송 중

- 배송 추적 카드 추가 (택배사, 운송장, 현재 상태)
- [배송 추적] 버튼
- 핑퐁이: "배송이 출발했어요! 예상 도착일: {날짜}"

### 4-4. 완료

- 전체 타임라인 표시 (딜 생성 → 오퍼 → 예약 → 결제 → 배송 → 도착)
- [리뷰 작성] 버튼
- 관전자 최종 결과 (포인트 적립 내역)

## 5. 실시간 업데이트

| 이벤트 | 화면 반응 |
|--------|----------|
| 새 오퍼 도착 | 오퍼 목록 최상단에 슬라이드 인, 가격 대시보드 갱신, "새 오퍼!" 토스트 |
| 관전자 예측 추가 | 관전자 수 카운트 업, 분포 차트 업데이트 |
| 참여자 추가 | 참여자 수 카운트 업 |
| 마감 임박 | 타이머 색상 빨간색 전환, 핑퐁이 긴급 메시지 |
| 딜 마감 | 상태 전환 애니메이션, 성사가 공개 |

MVP에서는 30초 간격 폴링으로 구현. 추후 WebSocket/SSE 전환.

## 6. 에러/빈 상태

| 상태 | 표시 |
|------|------|
| 딜 로딩 중 | 스켈레톤 UI (가격 영역 + 오퍼 3개 플레이스홀더) |
| 딜 없음 (404) | "존재하지 않는 딜이에요" + 홈으로 이동 버튼 |
| 오퍼 0개 | "아직 오퍼가 없어요. 판매자들이 준비 중!" + 핑퐁이 안내 |
| anchor_price 미도착 | 시장가 영역: "분석 중..." 스피너, 나머지 정상 표시 |
| 네트워크 에러 | "연결이 불안정해요. 다시 시도해주세요" + 재시도 버튼 |
